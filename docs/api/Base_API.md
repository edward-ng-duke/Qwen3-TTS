# Qwen3-TTS Base / Voice Clone API

Base URL: `http://localhost:4967`（默认；通过 `PORT` 环境变量改）

Base 变体用于根据参考音频克隆声音，并支持保存可复用的音色文件。启动时需要选择：

```bash
MODEL_VARIANT=base docker compose up -d
```

交互式 Swagger 文档：`http://localhost:4967/docs`

---

## 端点一览

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/v1/health` | 服务/模型就绪状态 |
| GET | `/v1/languages` | 支持语言列表 |
| POST | `/v1/clone` | 上传参考音频，一次性克隆并合成 |
| POST | `/v1/voice/save` | 上传参考音频，保存可复用音色 `.pt` |
| POST | `/v1/voice/generate` | 上传已保存音色 `.pt`，合成新文本 |

以下端点在 Base 变体下不可用，返回 `404`：

- `/v1/voices`
- `/v1/voices/{voice_id}/preview`
- `/v1/tts`
- `/v1/tts/stream`
- `/v1/audio/speech`
- `/v1/tts/design`

## 快速开始

```bash
# 启动 Base / Voice Clone
MODEL_VARIANT=base docker compose up -d

# 健康检查
curl -s http://localhost:4967/v1/health | jq

# 一次性克隆并合成
curl -X POST http://localhost:4967/v1/clone \
  -F text="This sentence is generated by the cloned voice endpoint." \
  -F language=English \
  -F ref_text="This is a clean reference voice sample for clone testing." \
  -F ref_audio=@ref.wav \
  -F response_format=wav \
  --output base_clone.wav

# 保存可复用音色
curl -X POST http://localhost:4967/v1/voice/save \
  -F ref_text="This is a clean reference voice sample for clone testing." \
  -F ref_audio=@ref.wav \
  --output base_voice.pt

# 使用保存后的音色生成新语音
curl -X POST http://localhost:4967/v1/voice/generate \
  -F text="This output uses the saved reusable voice prompt." \
  -F language=English \
  -F voice_prompt=@base_voice.pt \
  -F response_format=wav \
  --output base_generate.wav
```

## 元信息接口

### `GET /v1/health`

返回服务状态、模型路径和当前变体。

示例：

```json
{
  "status": "ok",
  "model_ready": true,
  "model_path": "/models/Qwen3-TTS-12Hz-1.7B-Base",
  "variant": "base"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `status` | string | `ok` 表示模型已加载，`loading` 表示仍在加载 |
| `model_ready` | boolean | 模型是否可推理 |
| `model_path` | string | 当前加载的模型路径 |
| `variant` | string | 当前应为 `base` |

### `GET /v1/languages`

返回模型支持的语言列表。`Auto` 表示由模型自动判断。

示例：

```json
{
  "languages": [
    "Auto",
    "auto",
    "chinese",
    "english",
    "german",
    "italian",
    "portuguese",
    "spanish",
    "japanese",
    "korean",
    "french",
    "russian"
  ]
}
```

## 一次性克隆并合成：`POST /v1/clone`

上传参考音频和参考文本，直接生成新文本的语音。

请求类型：`multipart/form-data`

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `ref_audio` | 是 | 参考音频文件，支持 `wav`、`mp3`、`flac` 等 soundfile 可解码格式 |
| `text` | 是 | 要合成的新文本 |
| `ref_text` | 条件必填 | 参考音频对应文字；`x_vector_only=true` 时可省略 |
| `x_vector_only` | 否 | 默认 `false`；`true` 表示只使用说话人向量，质量可能下降 |
| `language` | 否 | 默认 `Auto` |
| `response_format` | 否 | `wav`、`mp3`、`flac`、`pcm`，默认 `wav` |
| `seed` | 否 | 整数；用于尽量复现同一次生成 |
| `temperature` | 否 | 采样温度 |
| `top_k` | 否 | top-k 采样 |
| `top_p` | 否 | top-p 采样 |
| `repetition_penalty` | 否 | 重复惩罚 |
| `max_new_tokens` | 否 | 最大生成 token 数 |

示例：

```bash
curl -X POST http://localhost:4967/v1/clone \
  -F text="用克隆出来的音色读这段新文本。" \
  -F language=Chinese \
  -F ref_text="这是参考音频里说的内容。" \
  -F ref_audio=@ref.wav \
  -F response_format=wav \
  -F max_new_tokens=512 \
  --output cloned.wav
```

成功时返回二进制音频。`response_format` 与响应类型对应关系：

| `response_format` | `Content-Type` | 说明 |
|---|---|---|
| `wav` | `audio/wav` | WAV 容器，24kHz 单声道 |
| `mp3` | `audio/mpeg` | MP3 音频 |
| `flac` | `audio/flac` | FLAC 音频 |
| `pcm` | `audio/L16; rate=24000; channels=1` | raw PCM int16 little-endian |

常见错误：

| 场景 | 状态码 | 示例 |
|---|---:|---|
| 缺少 `ref_audio` | 422 | FastAPI 表单校验错误 |
| `text` 为空 | 400 | `{"detail":"text is required"}` |
| `x_vector_only=false` 且缺少 `ref_text` | 400 | `{"detail":"ref_text is required when x_vector_only is false"}` |
| 参考音频为空 | 400 | `{"detail":"ref_audio file is empty"}` |
| 参考音频无法解码 | 400 | `{"detail":"could not decode ref_audio: ..."}` |

## 保存可复用音色：`POST /v1/voice/save`

上传参考音频，返回一个可复用的 `.pt` 音色文件。

请求类型：`multipart/form-data`

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `ref_audio` | 是 | 参考音频文件 |
| `ref_text` | 条件必填 | 参考音频对应文字；`x_vector_only=true` 时可省略 |
| `x_vector_only` | 否 | 默认 `false` |

示例：

```bash
curl -X POST http://localhost:4967/v1/voice/save \
  -F ref_text="这是参考音频里说的内容。" \
  -F ref_audio=@ref.wav \
  --output my_voice.pt
```

响应：

- `Content-Type`: `application/octet-stream`
- `Content-Disposition`: `attachment; filename="voice.pt"`
- 内容是可被后续 `/v1/voice/generate` 使用的 torch `.pt` blob

常见错误：

| 场景 | 状态码 | 示例 |
|---|---:|---|
| 缺少 `ref_audio` | 422 | FastAPI 表单校验错误 |
| `x_vector_only=false` 且缺少 `ref_text` | 400 | `{"detail":"ref_text is required when x_vector_only is false"}` |
| 参考音频无法解码 | 400 | `{"detail":"could not decode ref_audio: ..."}` |

## 使用保存音色合成：`POST /v1/voice/generate`

上传 `/v1/voice/save` 生成的 `.pt` 文件，并用该音色合成新文本。

请求类型：`multipart/form-data`

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `voice_prompt` | 是 | `/v1/voice/save` 输出的 `.pt` 文件 |
| `text` | 是 | 要合成的新文本 |
| `language` | 否 | 默认 `Auto` |
| `response_format` | 否 | `wav`、`mp3`、`flac`、`pcm`，默认 `wav` |
| `seed` | 否 | 整数；用于尽量复现同一次生成 |
| `temperature` | 否 | 采样温度 |
| `top_k` | 否 | top-k 采样 |
| `top_p` | 否 | top-p 采样 |
| `repetition_penalty` | 否 | 重复惩罚 |
| `max_new_tokens` | 否 | 最大生成 token 数 |

示例：

```bash
curl -X POST http://localhost:4967/v1/voice/generate \
  -F voice_prompt=@my_voice.pt \
  -F text="用之前保存的音色说这句话。" \
  -F language=Chinese \
  -F response_format=wav \
  --output out.wav
```

常见错误：

| 场景 | 状态码 | 示例 |
|---|---:|---|
| 缺少 `voice_prompt` | 422 | FastAPI 表单校验错误 |
| `text` 为空 | 400 | `{"detail":"text is required"}` |
| `.pt` 为空 | 400 | `{"detail":"voice_prompt is empty"}` |
| `.pt` 无法加载 | 400 | `{"detail":"could not load voice_prompt: ..."}` |
| `.pt` 结构不正确 | 400 | `{"detail":"invalid voice_prompt format"}` |

## 推荐测试流程

Base 变体最小闭环测试应该覆盖三件事：

1. `/v1/clone` 能直接用参考音频生成语音。
2. `/v1/voice/save` 能生成非空 `.pt` 音色文件。
3. `/v1/voice/generate` 能复用该 `.pt` 继续生成语音。

如果没有现成参考音频，可以先用 CustomVoice API 生成一段参考音频：

```bash
MODEL_VARIANT=customvoice docker compose up -d

curl -X POST http://localhost:4967/v1/tts \
  -H 'Content-Type: application/json' \
  -d '{
    "text": "This is a clean reference voice sample for clone testing.",
    "speaker": "Ryan",
    "language": "English",
    "instruct": "Speak clearly and naturally.",
    "response_format": "wav",
    "sampling": {"max_new_tokens": 512}
  }' \
  --output ref.wav

docker compose down
```

然后切换到 Base 变体执行克隆/保存/复用。

- `MODEL_VARIANT=base docker compose up -d`
